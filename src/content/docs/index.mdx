## Why This Exists

The current game development loop is broken. You write code, switch to the editor, hit play, Alt-Tab back, read the console, fix a typo, repeat. Every context switch costs minutes and focus.

Aesthetic Engine eliminates the switch. Your AI agent becomes a first-class participant in the development loop: it launches the game, sees what's on screen, edits your source files, and verifies the fix — without you ever leaving the chat window.

This isn't a plugin that adds a button. It's infrastructure that makes your game observable and controllable by AI.

## Open Source Infrastructure

Aesthetic Engine is an open-source game development studio building agentic tooling for Godot. Both projects are MIT-licensed, publicly developed on [GitHub](https://github.com/Aesthetic-Engine), and designed to be forked, vendored, and extended without restriction.

The tools exist because the gap between "describe what you want" and "see it running" should be zero. Every component — the TCP bridge, the MCP layer, the wireframe renderer, the CRT pipeline — is built to close that gap.
