## Why This Exists

The current game development loop is broken by friction. You write code, switch to the editor, hit play, Alt-Tab back, read the console, fix a typo, repeat. Every context switch drains your focus.

Aesthetic Engine Builder and Godot Runtime Bridge eliminate the switch. Your AI agent becomes a highly capable technical assistant that handles the tedious develop → observe → verify loop. You stay in the flow of architecture and design while your agent launches the game, reads the live state, applies your directed changes, and verifies the fix — all without leaving the chat window.

Aesthetic Engine Builder works with Cursor to build 1983 microcomputer-style games: wireframe graphics generated by code, not diffusion models. Under your direction you can build and iterate rapidly, with technical constraints that keep API costs manageable.

It's sovereign infrastructure that makes your runtime observable and controllable — a truly unchained workflow.

- **Cursor-Native** — Built specifically for Cursor Agent workflows. MCP tools, click-to-copy prompts, screenshot verification — the entire develop → observe → verify loop runs from the chat window.
- **LLM-Optimized Docs** — Every documentation page is structured as flat, self-contained chunks — explicitly formatted for RAG ingestion and AI copilot scraping.
- **MIT Licensed** — Fork it, vendor it, ship it inside your game. No usage restrictions, no telemetry, no lock-in.
- **Security-First** — Localhost-only binding, token auth, capability tiers, eval disabled by default. The bridge is inert in production builds.

## Open Source Infrastructure

Aesthetic Engine is an open-source game development studio building agentic tooling for Godot. Both projects are MIT-licensed, publicly developed on [GitHub](https://github.com/Aesthetic-Engine), and designed to be forked, vendored, and extended without restriction.

The tools exist because the gap between "describe what you want" and "see it running" should be zero. Every component — the TCP bridge, the MCP layer, the wireframe renderer, the CRT pipeline — is built to close that gap.
